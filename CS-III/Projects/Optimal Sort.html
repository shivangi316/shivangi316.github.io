the explanations to the graphs

Link to the graphs


each of the nine graphs represent ine section of data. they are seperated so that you can veiw and easily compar each of the nine types of files. the graphs represent the correlation between size of file and execution time. overall, Bubble sort had the slowest execution times, and largest rate of change depicted in the graphs. each type of file took longer to execute, ordered files were the quickest because they were already ordered and therefore did not have to parse through ultiple scenarios. the random files took the longest because each index had to be parsed and sorted. the graphs show the correlation between the increase in the size of the file and the length of time each execution took.
